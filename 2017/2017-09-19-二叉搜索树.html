二叉搜索树

发表于 2017-09-19 | 分类于 算法 |
字数统计 1,512 | 阅读时长 5
博客摘要：本文主要总结了与二叉查找树相关的知识点。

二叉搜索树

二叉搜索树(二叉查找树/二叉排序树)(Binary search Tree)

定义：一棵二叉树，若满足如下条件，即任何结点的键值一定大于其左子树中的每一个结点的值，并小于其右子树中的每一个结点的值。则该二叉树为二叉搜索树。二叉搜索树提供对数时间的插入和访问效率。

最小值：二叉搜索树中最左侧的结点，即为最小值。即沿根结点一直递归访问左子树，直到树的末端，最末端的结点即为所求。
最大值：二叉搜索树中最右侧的结点，即为最大值。原理同上。
插入操作: 插入新元素时,可从根结点开始,若新元素键值比当前结点的键值大,则进入当前结点的左子树;若新元素的键值比当前结点的键值大,则进入当前结点的右子树。如此递归，直到尾端，即为插入点。
删除操作: 删除操作分为两种情况，情况如下：

  1. 若待删除结点A只有1个子结点，则直接将A的子结点连接至A结点的父节点，并将A结点删除即可。
  2. 若待删除结点A有两个子结点，则以右子树内的最小结点取代A结点即可。

平衡二叉搜索树(balanced binary search tree)

定义：若一个二叉搜索树是平衡的，则该二叉树为平衡二叉搜索树。“平衡”的大致意义是没有任何一个结点过深。不同的平衡条件，具有不同的效率。根据平衡标准的不同，有AVL-tree,RB-Tree,AA-tree树。

自平衡二叉搜索树(AVL-tree,Adelson-Velskii-Landis tree)

定义：若一棵二叉搜索树中，任何一个结点的左子树的高度与右子树的高度之差的绝对值小于等于1，则这颗二叉搜索树为自平衡二叉搜索树。

AVL-tree平衡调整

当插入一个结点导致AVL树不平衡时。我们需要找到平衡状态被破坏的各个结点中最深的那个结点。我们将该结点记为X。

方法，根据插入点所处位置的结构特点分为如下4种情形：

  1. 插入结点位于X的左结点的左子树—左左
  2. 插入结点位于X的左结点的右子树—左右
  3. 插入结点位于X的右结点的左子树—右左
  4. 插入结点位于X的右结点的右子树—右右

说明：

  • 情况1,4彼此对称，称为外侧(outside)插入，可以使用单旋转(single rotation)操作调整解决。
  • 情况2,3彼此对称，称为内侧(inside)插入，可以使用双旋转(double rotation)操作调整解决。

红黑树(RB-Tree,Red-Black Tree)

红黑树是一棵平衡二叉查找树，其定义如下：

  1. 每个结点不是红色就是黑色；
  2. 根结点为黑色；
  3. 如果结点为红色，其子结点必须为黑色(父子结点不能同时为红)；
  4. 任一结点至NULL(树尾端)的任何路径，所含黑结点必须相同。
  5. NULL结点为黑色
    调整条件：根据规则4，新结点必须为红色；根据规则3，新结点的父节点必须为黑色。当一个新结点根据二叉查找树的规则到达其插入点时，若未能符合上述条件时，就必须调整颜色并旋转树形。
