# 进程管理
## 1.进程与线程的区别

+ 定义：线程是一个程序的独立执行单位。而线程是一个进程的一个子集。
+ 访问数据段的方式：每个进程都会有其父进程数据段的一个拷贝；而线程可以直接访问所属进程的数据段。
+ 通信方式：进程之间必须使用进程间通信的方式与其他进程通信。而线程可以直接与所属进程中的其他线程通信。
+ 开销：进程具有相当大的开销；而线程几乎没有什么开销。
+ 创建：创建新的进程需要复制父进程；而线程创建很简单。
+ 控制：进程只能控制器子进程；而同一个进程中线程可以相互控制(kill，等待其他对等线程终止)。
+ 改变：父进程的任何改变不会影响到子进程；任何主线程中的改变都会影响同一进程中的其他线程。
+ 内存：进程运行在独立的内存空间；线程运行在一个共享的内存空间。
+ 文件描述符：大部分的文件描述符无法再进程间共享；而线程共享文件描述符。
+ 文件系统：进程之间不共享虚拟地址空间；而同一进程中的线程共享该进程的虚拟地址空间的整个内容。
+ 信号：进程之间不共享信号；而线程之间共享信号。
+ 控制方式：进程由操作系统控制；而线程由程序设计者控制；
+ 独立性：进程是不独立的；而线程是独立的。

## 2.线程

   + 线程独有的资源
     每个线程都有它自己的线程上下文(thread text)，包括一个唯一的线程ID、栈、栈指针、程序计数器、通用寄存器和条件码。所有运行在一个进程里的线程共享该进程的整个虚拟地址空间。
   + 线程共享的资源
     进程的所有信息对该进程的所有线程都是共享的，包括可执行的程序文本，程序的全局变量和堆内存，栈以及文件描述符。

## 3. 进程间通信(Inter-Process Communication)
进程间通信是指两个进程或线程间传送数据或信号的技术或者方法。每个进程都有自己的一部分独立的系统资源，彼此是隔离的。为了能够使不同的进程互相访问资源并进行协调工作，就需要在进程之间进行通信。
IPC的优点：
+ 信息共享：多个进程之间可以共享文件。

### 进程间通信方式
#### 1. 信号

信号是Unix/Linux系统在一定条件下生成的事件。信号是一种异步通信机制，进程不需要执行任何操作来等待信号的到达。信号异步通知接收信号的进程发生了某个事件，然后操作系统将会中断接收到信号的进程的执行，转而去执行相应的信号处理程序。

**信号的特点**：

+ 可携带的信息极少。不适合需要经常携带数据的通信
+ 不具备同步机制，类似于中断

#### 2.信号量(semaphore) 
   信号量是一个计数器。信号量用于实现进程间的互斥和同步，而不是用于存储进程间通信数据。
　　

**信号量的特点**：

  + 信号量用于进程间同步，若要在进程间传递数据需要结合内存共享。
  + 信号量基于操作系统的PV操作，程序对信号量的操作都是原子性的。
  + 每次对信号量的PV操作不仅限于对信号量值得加1或减1，而且可以加减任意正整数。
  + 支持信号量组。

**说明** ：最简单的信号量是只能去0或1的变量，这是最常见的形式，叫做二值信号量(binary semaphore)。而可以取多个正整数的信号量被称为通用信号量。

#### 3.(匿名)管道

   管道，通常指无名管道，是UNIX系统中最古老的形式。例如：`ls | wc -l` 

   管道的特点：

   + 它是半双工的(即数据只能在一个方向流动)，具有固定的读端和写端。
   + 他只能用于具有亲缘关系的进程之间通信(父子进程或者兄弟进程)。
   + 它可以看成一种特殊的文件，对于它的读写也可以使用普通的read()、write()等函数。但是它不是普通的文件，并不属于其他任何文件系统，并且只存在于内存中。
   + 自身具备同步机制

**说明**：当一个管道建立时，它会创建两个文件描述符：fd[0]为读而打开，fd[1]为写而打开。

```c
#include <stdio.h>
// a call to the pipe() function returns a pair of file descriptors. One of // // these descriptors is connected to the write end of the pipe, and the other  // is connected to the read end.
pipe();
```

#### 4.命名管道(FIFO, First In-First Out)
  FIFO，也称为命名管道，它是一种文件类型。

**命名管道的特点**：
+ FIFO可以在无关的进程之间交换数据，与无名管道不同。
+ FIFO有路径名与之关联，它以一种特殊设备文件的形式存在于文件系统中。
  说明：FIFO的通信方式类似于在进程中使用文件来传输数据，只不过FIFO类型文件同时具有管道的特性。在数据读出时，FIFO管道中同时清除数据，并且是“先进先出”的。
+ 相互之间没有关联的进程都可以打开(使用系统调用open())管道，然后进行数据读写操作。
+ 自身具备同步机制

**说明**：
```cpp
//create a FIFO and read,write as a nomal file
mknod("myfifo", s_IFIFO | 0644, 0); 
```

#### 5.共享内存
  共享内存(shared memory)是指两个或者多个进程共享一个给定的存储区。

**共享内存的特点**：
+ 共享内存速度比较快，因为进程直接对内存进行存取
+ 因为多个进程可以同时操作，因此需要同步
+ 信号量和共享内存通常结合使用，信号量用来同步对共享资源的的访问。

#### 6.消息队列
  消息队列是消息的链接表，存放在内核中。一个消息队列中由一个标识符(即队列ID)来标识。

**消息队列的特点** ：

+ 消息队列是面向记录的，其中的消息具有特定的格式以及特定的优先级。
+ 消息队列独立于发送和接收进程。进程终止时，消息队列及其内容不会被删除。
+ 消息队列可以实现消息的随机查询，消息不一定要以先进先出的次序读取，也可以消息的类型读取。
+ 具备同步机制

#### 7.套接字
套接字(Socket)是由Berkeley在BSD系统中引入的一种基于连接的IPC，是对网络接口(硬件)和网络协议(软件)的抽象。
套接字的特点：
+ 既解决了无名管道只能在相关进程间单向通信的问题，又解决了网络上不同主机之间无法通信的问题。
+ 该方式自身携带同步机制，不需要额外的方式来辅助实现同步。
+ 随进程持续

#### 10.文件描述符
在C语言中，我们经常会使用fopen()，fclose()，fwrite()等文件操作函数。这些函数中都会有一个`FIFE *`。这实际上是有高级别函数实现的文件描述符，它们使用了诸如create(), open(), write(), close()等系统调用。文件描述符其实就是类似于stdio.h头文件中`FILE *`的一个整型数。例如stdin的文件描述符为“0”,stdout为“1”，stderror为“2”。任何一个文件都可以使用fopen()函数来获取它的文件描述符。

# 内存管理

## 静态链接与动态链接

静态链接的特点

1. 静态链接方式中整个函数库在编译时都会被整合进目标代码中。因此编译后的执行程序不需要外部的函数库的支持。但是若其中某个函数库发生改变，需要重新编译程序。
2. 代码更精简，不需要做版本检查。
3. 程序分发时，文件个数少，因为所有函数都静态链接到源文件中了。程序中只包含编译需要的函数，而不需要整个库。
4. 生成的可执行文件更大。
5. 重复的库可能出现在多个进程中，造成资源浪费

动态链接库的特点

1. 动态函数库在编译时并没有编译进目标代码中。当程序执行到对应的动态函数是才调用该函数库里对应的函数。因此，动态链接方式生成的可执行文件相对较小。
2. 由于动态函数库没有在编译时整合到可执行程序中，而是在程序运行过程中动态的申请并调用，所以程序的运行环境中必须提供相应的库。
3. 动态链接方式中，动态函数库的改变不会影响到可执行程序，因此动态链接方式便于程序升级。
4. 某个程序在运行时要调用某个动态函数库函数时，OS首先检测对应的动态库是否已经被其他进程加载到内存中。若要使用到的库已经被加载到内存中，直接使用即可。若未被加载，才会被载入。这样做的优点就是节省内存空间。
5. 若要升级某个软件，而该软件改变的部分在动态链接函数中，那只需要改变动态链接函数而不需要改变软件的其他部分。因此，升级方便。

# 工具使用
## GDB调试多进程的方法

   + 单独调试子进程

     子进程从本质上说也是一个进程，因此我们可以用通用的GDB调试方法来调试它。方法如下：首先我们先运行父进程，然后找到目标子进程的PID，再将其附加(attache)到GDB调试器上。

   + 使用调试器选项follow-fork-mode

     gdb调试器的选项follow-fork-mode允许我们选择程序在执行fork系统调用后是继续调试父进程还是调试子进程。其方法如下：`set follow-fork-mode mode` 。其中，mode的可选值时parent和child，分别表示调试父进程和子进程。
##  GDB调试多线程的方法

   gdb有一组命令可辅助多线程程序的调试。下面仅列举出其中常用的一些：

   + info threads    显示当前可调试的所有线程。gdb为每个线程分配一个ID，我们可以使用这个ID来操作对应的线程。ID前面有“*”号的线程是当前被调试的线程。
   + thread ID    调试目标ID指定的线程
   + set scheduler-locking [off|on|step]    调试多线程程序时，默认除了被调试的线程在执行外，其他线程也在执行。但有时候我们希望只让被调试的线程运行。这可以通过这个命令来实现。该命令设置scheduler-locking的值：off表示不锁定任何线程，这是默认值；on表示只有当前被调试的线程会继续执行；step表示在单步执行时，只有当前线程会执行。

### 使用strace， pstack

   strace：打印一个程序和它的子进程调用的每个系统调用的轨迹。用-static编译程序，能得到一个更清楚的轨迹，而不带有大量与共享库相关的输出。

   pstack：显示每个进程的栈跟踪。pstack命令必须由相应进程的属主或 root 运行。可以使用 pstack 来确定进程挂起的位置。
